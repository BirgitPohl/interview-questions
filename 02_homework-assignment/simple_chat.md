## Goal
Build a minimal chat UI that talks to OpenAI or any other large language model with similar
ChatCompletion data structure via a server-side route inside your frontend app
(e.g., Next.js/Remix/SvelteKit). Keep it clean, small, and standards-compliant.

## Scope
- Single repo, single app (no separate backend service).
- One server-side API route in the app: /api/chat
- Reads LLM_API_KEY from environment.
- Accepts an array of chat messages and returns a reply.
- No secrets in the browser.

### Simple chat UI
- Input + “Send”.
- Renders a scrollable thread of user/assistant messages.
- Disable “Send” + show a loading state while waiting.

## Quality bar
- Excellent code design.
- Linting + formatting (your stack’s standard).
- Sensible file structure and small functions.
- Type safety if available (TypeScript encouraged and preferred).
- One meaningful test.
- Example ideas: route rejects missing body, or reducer formats messages.
- Open up a CodeSanbox
- Use CodeSanbox and send us the link to your sandbox. Make sure that we can see the code and run the app.
